## 1. Вступ
Sorokchat - це інформаційна система призначена для, безпосереднього, обміну повідомлень між користувачами. 
Ця інформаційна система повинна забезпечувати такі вимоги як:
- конфіденційність;
- цілісність;
- доступність.

Дана інформаційна система використовує схему криптографічного захисту E2EE.

E2EE (End-to-End Encryption) або наскрізне шифрування — це метод криптографічного захисту, за якого дані шифруються на пристрої відправника і розшифровуються лише на пристрої отримувача. 
Це гарантує, що провайдери зв'язку, хакери чи сервери не зможуть перехопити або прочитати повідомлення.

## 2. Функціональні вимоги до системи

### 2.1. Користувачі

У цій інформаційній системі є користувачі. 
Користувач має змогу зареєструватися у системі, увійти у систему, отримати данні про себе, вийти з системи.
У користувача наявні такі поля:
- логін - потрібен для ідентифікації користувача у системі;
- видиме ім'я - це опціональне поле, котре використовується для відображення імені користувача, котре відмінне від логіну, якщо ж це поле не вказане, то для відображення імені використовується логін;
- пароль - поле котре, використовується для аутентифікації користувача у системі;
- роль - поле, яке використовується для авторизації користувача у системі. На основі ролі, користувачу будуть надаватися відповідні права у системі.

### 2.2. Чати

Чати - це сутність в інформаційній системі, яка зберігає данні про конкретний чат.
