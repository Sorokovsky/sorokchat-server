# Sorokchat - Український Месенджер

Цілю створення серверної програми, далі "Sorokchat Server", є забезпечення роботи захищеного месенджера.

Sorokchat Server повинен забезпечувати роботу з користувачами, чатами та повідомленнями.

Кожна сутність у системі має ідентифікатор, дату створення та дату оновлення. Винятком є повідомлення, що не містить
ідентифікатора.

## Користувачі

Користувач - це сутність, яка містить унікальне ім'я користувача(латинецею), ім'я для відображення, пароль(зберігається
виключно у виглядів результату хеш-функції), номер телефону та електронну адресу.

Користувачам має бути забезпечена можливість: зареєструватися, авторизуватися та вийти з облікового запиту.
Авторизація повинна відбуватися через Access Token, та Refresh Token.
Користувач має право отримати данні про себе, редагувати данні про себе та видалити свій акаунт.

## Access Token

Access token - це JWS токен, який зберігає мінімальні данні про користувача і дає можливість користувачу авторизуватися
у системі.
Термін дії Access token's не може перевищувати 30 хвилин.

## Refresh Token

Refresh token - це JWE токен, який зберігає ідентифікатор Access-token і дає можливість оновити Access token та Refresh
token.
Термін дії Refresh token не може перевищувати 7 календарних днів.

## Чати

Чат - це сутність, яка містить у собі користувачів. Кожен користувач у чаті є рівноправним.

## Контакт

Контакт - це чат, який має рівно 2 користувачі.

Контакт не може мати назву, вона(тобто назва чату) визначається динамічно на клієнті залежно від авторизованого
користувача.
Хоч контакт має у собі користувачів, але забороняється додавання нового чи вилучення користувачів.

## Група

Група - це чат, який має назву та опис. Група має необмежену кількість користувачів.
